# 风险管理

## 概述

本项目风险按以下四类管理：

- 技术风险（平台依赖、API 集成、安全、性能）
- 资源风险（人员、预算、基础设施）
- 进度风险（任务依赖、外部依赖、技术难点）
- 市场与商业风险（竞争、政策、用户接受度、合规）

## 风险评估

采用风险矩阵：`风险值 = 可能性(1-5) × 影响(1-5)`。

- 低（1-4）：接受/监控
- 中（5-9）：减轻/监控
- 高（10-15）：规避/转移/减轻
- 极高（16-25）：立即处理

## 重点风险（示例）

- SecondMe 平台稳定性：通过本地缓存、降级策略、可用性监控缓解
- 凭证泄露：最小权限、严格权限控制、日志脱敏、轮换密钥
- 关键路径延误：关键任务缓冲、阻塞问题日清、并行化交付
